# テキストフロー品質ゲート

テキストフロー生成後、ユーザーに提示する前に全項目を検証する。
1つでも違反があれば修正してからユーザーに提示する。

---

## G-1: フォーマット準拠

- [ ] メタ情報（フロー名/目的/粒度/レーン）が全て記載されているか
- [ ] 目的が「〜が完了するまで」で終わっているか
- [ ] フロー構造が標準フォーマット（text-flow-template.md）に準拠しているか
- [ ] 分岐番号が通し番号で、飛びがないか

---

## G-2: 分岐設計

- [ ] 全分岐がYes/Noで答えられる問いになっているか
- [ ] 全分岐でYes↓が正常系・多数派になっているか
- [ ] 問いに疑問符「？」が含まれていないか
- [ ] 【分岐ラベル】に接続元（「分岐①のNo→」等）が明記されているか
- [ ] No→を先に書き、Yes↓を下に続ける書き方になっているか
- [ ] 「やるかスキップか」分岐の場合、text-flow-template.mdの判定フロー（3段階）に従っているか
- [ ] No→側にL字合流線（右→下→左）が発生する構造になっていないか

---

## G-3: 粒度の一貫性

- [ ] 全ノードが粒度マスターの選択列と一致しているか
- [ ] 粗いノードと細かいノードが混在していないか
- [ ] 経営層向けで「ログインする」等の操作レベルがないか
- [ ] エンジニア向けで「会計処理」等の概要レベルがないか

---

## G-4: 3要素チェック（業務担当向け以上）

- [ ] 全ノードに「何をする」が動詞で記載されているか
- [ ] レーンが定義されている場合、各ノードの「誰が」がレーンで表現されているか
- [ ] 「何で」（実施場所/システム）が補足テキストで記載されているか（推奨）

---

## G-5: 曖昧表現の検出

以下の表現がノード内に含まれている場合、要確認事項に追加する：

- 「〜等」「〜など」→ 具体的な列挙が必要
- 「仮」「仮：」→ 未確定であることを要確認事項に記載
- 「未定」「TBD」「要検討」→ 同上
- 「〜かもしれない」「〜の可能性」→ 確定が必要
- 「適宜」「必要に応じて」→ 条件の明確化が必要

---

## G-6: 構造の完全性

- [ ] 開始ノードがあるか
- [ ] 全ての分岐のYes/No両方に行き先が記載されているか
- [ ] 全てのサブフローに「完了」または他のフローへの合流が記載されているか
- [ ] 孤立したノード（どこからも接続されない）がないか

---

## G-7: 設計判断メモ

- [ ] 設計判断メモが記載されているか
- [ ] 空の場合、なぜ空かが記載されているか
- [ ] 参照ノードがある場合、その採用理由が記載されているか
- [ ] 分岐の向きの選択理由が記載されているか

---

## G-8: 要確認事項

- [ ] G-5で検出した曖昧表現が全て要確認事項に含まれているか
- [ ] ノード内の実施場所が不明確な場合、要確認事項に含まれているか
- [ ] 要確認事項に通し番号が付いているか
