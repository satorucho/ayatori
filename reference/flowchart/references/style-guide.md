# スタイルガイドライン（表現ルール）

座標計算、スペーシング、図形サイズ、矢印パターン、カラーパレットの定義。
生成時の座標決定は**すべてこのファイルのルールに従う**。

## 3. 矢印パターン

### パターンM

| 方向 | 間隔 | 用途 |
|------|------|------|
| 垂直 | 64px | ノード間の直結、分岐Yes↓、通常の縦接続すべて |
| 水平 | 80px | 分岐No→短い枝 |

### パターンS（合流専用）

| 方向 | 間隔 | 用途 |
|------|------|------|
| 垂直 | 24px | 合流点→直後のノードのみ |

パターンSは合流点→直後のノード以外で使用禁止。

### パターンL（可変長）

時間軸整合用、列展開の水平矢印。間隔は整合ポイントや列のCXに応じて可変。

**パターンLの最小ノード端間隔: 84px（描画長64px）。これ未満は禁止。**

### パターン選択ルール

1. デフォルトはパターンM
2. パターンSは合流点→直後のノードにのみ使用
3. パターンLは時間軸整合と列展開のみ（最小84px厳守）
4. 同一フロー内で同種の矢印は同じパターン・同じ長さ

### 矢印ギャップ

| パラメータ | 値 | 適用箇所 |
|-----------|------|---------|
| 始点ギャップ | +6px | 前ノードの下端/右端から |
| 終点ギャップ | -6px | 次ノードの上端/左端まで |
| marker補正 | -8px | 終点のみに追加適用 |

```
垂直矢印:
  y1 = 前ノード下端 + 6
  y2 = 次ノード上端 - 6 - 8 = 次ノード上端 - 14

水平矢印:
  x1 = ノード右端 + 6
  x2 = 次ノード左端 - 6 - 8 = 次ノード左端 - 14
```

重要：座標表の上端/下端/左端/右端は図形の端。ギャップとmarker補正はSVG生成時のline要素にのみ適用する。座標表にギャップを含めてはならない。

## 4. フロー方向の原則

- メインフロー: 左側を上→下に直進
- Yes↓: 下方向に直進
- No→: 右方向に展開
- 合流線にはmarker-endを付けない

### 4-1. 分岐合流の計算ルール

**合流Y座標の計算式:**
```
合流Y = Yes側の最終ノード下端 + 64px（パターンM垂直）
```

**No→の合流線X座標:**
```
No→の縦線のX座標 = Yes側の全ノードの最大右端 + 30px以上
```

No→の合流線がYes側のrect/polygon/ellipseの内部を通過してはならない。
合流Y座標は目測ではなく必ず計算で求め、計算表の合流Y列に記載する。

### 4-2. 整合ポイント再検証（2レーン以上の場合）

座標表のStep 2〜3で分岐のNo→側にノードが追加された結果、新たな整合関係が生まれていないか再検証する。

**特定基準（いずれかに該当すれば整合ポイント）:**
- A) 同一の業務イベント: 同じ名前、または同義の処理
- B) 送受信の関係: 一方が送り、他方が受ける処理

同じ時刻というだけでは整合ポイントにしない。迷ったらユーザーに確認する。

## 5. 列（Column）構成ルール

分岐No→の先に2ステップ以上の処理がある場合、新しい列が生まれる。

### 用語
- 列0: メインフロー
- 列1以降: 分岐No→から展開されるサブフロー
- 短い枝: 分岐No→の先が1ステップで完結する処理。列を生成しない

### 1列1接続原則
列0から列Nへの接続は、1つの分岐からのみ行う。複数の分岐から同じ列に接続することを禁止する。必要な場合は参照ノード（同色）で表現する。

### 列間の干渉防止
全列のY座標確定後、以下を検証する：
- 列0の短い枝が列1のノードと横方向で重なっていないか
- 列0から列1への水平矢印が列1の既存ノードを横切っていないか

干渉が検出された場合、列のCXを右にずらす。

### ネスト上限
列は最大3列（列0〜列2）まで。3列を超える場合はフロー構造を見直す。

## 6. 図形サイズの自動計算ルール

SVGを書く前に、全ノードの図形サイズを計算表で確定させる。計算表なしにSVGを書くことを禁止する。

### 6-1. 定数

| パラメータ | 値 |
|-----------|------|
| 全角文字幅 | フォントサイズ × 1.05 |
| 半角文字幅 | フォントサイズ × 0.6 |
| 行の高さ | フォントサイズ × 1.5 |
| 左右パディング | 24px |
| 上下パディング | 10px |

### 6-2. テキスト改行ルール
1行のテキスト幅が矩形幅-48px（左右パディング）を超える場合、改行する。
改行の優先切れ目：助詞の直後 → 読点の直後 → 括弧の直前 → スラッシュの前後

### 6-3. テキスト幅の計算
各行: 行幅 = 全角文字数 × 全角文字幅 + 半角文字数 × 半角文字幅
複数行: テキスト幅 = max(各行の行幅)
テキスト高さ = Σ(各行のフォントサイズ × 1.5)

### 6-4. 図形サイズの計算

**矩形:**
  幅 = テキスト幅 + 48（左右パディング×2）
  高さ = テキスト高さ + 20（上下パディング×2）

**ダイヤモンド:**
  Step A: 内接幅 = テキスト幅 + 48、内接高さ = テキスト高さ + 20
  Step B: W = ceil(内接幅 / 2 × 1.42)、H = ceil(内接高さ / 2 × 1.42)
  Step C: points = "CX,CY-H CX+W,CY CX,CY+H CX-W,CY"
  **ダイヤモンドテキストは最大2行×8全角文字。** 収まらない場合はテキストを短縮する。短縮はユーザーに確認してから実行。

**楕円:**
  主テキスト+補足テキストを全て楕円内に収める。補足テキストがある場合、ryを拡大して2行分を確保する。
  rx = max(主テキスト幅, 補足テキスト幅) / 2 × 1.2 + 24
  ry = テキスト高さ合計 / 2 + 10
  **楕円テキストは最大2行。** 3行以上は禁止。テキストを短縮するか、矩形に変更する。
  ※ 楕円は端に近づくほど使える幅が狭くなるため、rxに×1.2の係数を適用。

### 6-5. 最小サイズ
| 図形種別 | 最小幅 | 最小高さ |
|---------|-------|---------|
| 矩形（メインフロー） | 220px | 40px |
| 矩形（短い枝） | 100px | 36px |
| ダイヤモンド | W=110, H=50 | - |
| 楕円 | rx=50, ry=20 | - |

### 6-6. テキスト凍結ルール
計算表に記載したテキストは凍結される。SVG生成時に計算表と異なるテキストを使用することを禁止する。テキストを変更したい場合、計算表に戻って再計算する。1文字の修正でも例外なく計算表からやり直す。

### 6-7. 同一列内の幅統一ルール
同一列内の矩形幅を最大値に統一する。ただし最大値との差が100px以上ある矩形は個別の計算値を使用する。ダイヤモンドは統一対象外。

## 7. 計算表のフォーマット（必須）

SVGを書く前に以下の3表を作成する。3表なしにSVGを書くことを禁止する。

### 7-1. テキスト計算表
| # | テキスト各行 | px | 全角 | 半角 | 行幅 | 行高 |

全ノード（短い枝を含む）のテキストを列挙する。
ノード列挙チェックリスト：
- [ ] メインフロー（列0）の全ノード
- [ ] 列1以降の全ノード
- [ ] 全分岐のNo→短い枝
- [ ] 開始/終了楕円

### 7-2. 図形サイズ表
| # | 種別 | テキスト幅 | テキスト高 | 計算幅 | 計算高 | 最小適用 | 確定幅 | 確定高 | W | H |

### 7-3. 座標表
| # | 種別 | 所属 | CX | CY | 上端 | 下端 | 左端 | 右端 | 合流Y |

「所属」列: 列0/列1/列1短枝/企業管理者 等。短い枝は必ず独立した行として含める。
**「合流Y」列は必須。** 分岐がある場合、合流Y = Yes側最終ノード下端 + 64px を記載する。分岐がないノードは空欄。
次ノードの上端 = 前ノードの下端 + 64（M垂直）。必ず前ノードの「下端」列を参照する。CYからの逆算を禁止する。

## 8. 境界ボックス管理

### 8-1. 境界ボックス計算表（必須）
座標表が完成したら作成する。
| レーン/列 | 最小左端 | 最大右端 | 最小上端 | 最大下端 |
全ノード（短い枝を含む）の左端/右端/上端/下端のmin/maxを算出する。

### 8-2. レーン配置のロジック（ノード配置先行方式）

以下の順序で計算する。仮決めを禁止する。

```
Step 1: ノードCXを決める
  最初のレーン: CX = 左余白40 + 最大矩形幅/2
  以降のレーン: CX = 前レーンの最大右端 + 80(余白) + 当レーン最大矩形幅/2

Step 2: 全ノード（短枝含む）の座標を確定する
  ⚠ 前提: Step 1が完了していること（全CXが確定）
  ⚠ CXが未確定のままStep 2に進むことを禁止

Step 3: レーン区切り線を決める
  ⚠ 前提: Step 2が完了していること（座標表の全行が埋まっていること）
  ⚠ 座標表の全行が埋まっていない状態でStep 3に進むことを禁止

  区切り線X = (前レーンの最大右端 + 次レーンの最小左端) / 2
  左右余白の差が50px以上になる場合、短枝テキストの短縮を検討する。

Step 4: 黒帯を配置する
  黒帯rect = 区切り線+30 〜 次の区切り線-30
  黒帯テキスト = 区切り線間の中央（ノードCXとは独立）

Step 5: SVG幅/高さを決める → 最終走査で確定
```

### 8-3. 修正時のルール

**座標の部分修正（文字列置換）を禁止する。**
座標を変更する場合、Step 1からゼロで再生成する。
**この禁止事項に対する例外・正当化・効率性の主張はすべて無効。例外なし。**

座標表のいずれかの値を修正した場合：
1. 境界ボックス計算表を再作成
2. レーン区切り線・SVG幅/高さを再計算
3. 影響を受ける全ノードの座標を更新
4. viewBox/width/height属性を更新

1箇所の修正でも全4ステップを実行する。短枝のテキスト/幅変更時もレーン区切り線を再計算する。

## 9. 最終走査（SVG生成の最終ステップ）

SVG内の全要素を書き終わった後、以下を実行する。省略不可。

1. 全rect要素から x + width の最大値を抽出
2. 全polygon要素から全頂点のx座標の最大値を抽出
3. 全ellipse要素から cx + rx の最大値を抽出
4. 上記1〜3の最大値 = 全要素最大右端
5. 同様に全要素の最大下端を算出
6. SVG幅 = 全要素最大右端 + 40（10px単位に切り上げ）
7. SVG高さ = 全要素最大下端 + 40（10px単位に切り上げ）
8. viewBox, width, height の3属性を全て更新
9. **レーン干渉チェック（レーンが2つ以上ある場合、必須）:**
   - 各レーン内の全要素の右端がレーン区切り線Xを超えていないか
   - 区切り線X = (前レーン最大右端 + 次レーン最小左端) / 2 になっているか
   - レーン区切り線を挟んで左右の余白が均等か

事前の境界ボックス計算表の値と最終走査の値が異なる場合、最終走査の値を採用する。
部分修正でも最終走査は省略不可。過去のSVG幅を参照せず毎回ゼロから走査する。
最終走査なしにpresent_filesを呼ぶことを禁止する。

## 10. Phase帯レイアウト（オプション）

Phase帯を使用する場合のみ適用する。使用しないフローではこのセクションを無視する。

### 10-1. レイアウト構成
```
Phase帯: 左端x=40, 幅=200px, 高さ=30px
テキスト: x=55（左寄せ, text-anchor=start）
フロー本体: Phase帯の右端 + 34px以上の間隔を確保
```

### 10-2. Phase帯幅の調整
デフォルト幅は200px。**テキスト長の目安:** 全角10文字以内。
テキストの全角文字数×16px×1.05がPhase帯幅を超えないことを確認する。

### 10-3. Phase帯のスタイル
```
rect: fill="#f0f0f0" stroke="#999" stroke-width="1" rx="3"
text: font-size="16" font-weight="600" fill="#555"
```
