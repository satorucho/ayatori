# Flowchart Skill

業務フローチャートの設計・HTML/SVG生成・セッション引き継ぎを行う統合スキル。
7つの参照ファイルを持ち、モードに応じて必要なファイルを読む。

## 参照ファイル

| ファイル | 役割 | 読むタイミング |
|---------|------|--------------|
| `references/granularity-master.md` | 粒度マスター表・選択フロー・深掘り基準 | 設計モード |
| `references/text-flow-template.md` | テキストフロー標準フォーマット・記法・出力例 | 設計モード |
| `references/quality-gate.md` | テキストフロー品質ゲート（8カテゴリ） | 設計モード |
| `references/template.md` | HTML/SVGの骨格・ノードパーツ・凡例 | 描画モード |
| `references/style-guide.md` | 図形サイズ計算・座標計算・矢印・境界ボックス・最終走査 | 描画モード |
| `references/quality-check.md` | SVG品質チェック（5カテゴリ27項目） | 描画モード |
| `references/handover-template.md` | handover.mdのテンプレート | 引き継ぎモード |
| `references/page-split.md` | ページ分割の手法（CSS・SVG分割・Y座標シフト） | 描画モード（ユーザー希望時のみ） |

## モード判定

ユーザーの指示から自動判定する。判定に迷う場合はユーザーに確認する。

### モードA: 設計モード

**トリガー**: 「業務フローを整理して」「フロー構造を考えて」「テキストフローを作って」、またはテキストフローなしでフロー図を依頼された場合

**読むファイル**: `granularity-master.md` → `text-flow-template.md` → `quality-gate.md`

### モードB: 描画モード

**トリガー**: 「フローチャートを作って」「HTMLにして」「SVGにして」+ テキストフローあり。または「レビューして」「修正して」+ 既存HTMLファイル

**読むファイル**: `template.md` → `style-guide.md` → `quality-check.md`

### モードC: 引き継ぎモード

**トリガー**: 「今日はここまで」「引き継ぎを作って」「ハンドオーバー」「中断する」

**読むファイル**: `handover-template.md`

## モードA: 設計モード

業務説明からテキストフロー.mdを生成する。HTML/SVG生成は行わない。

### A-0: 入力判定と粒度合意（省略禁止）

ユーザーの入力を判定する：

- **テキストフロー提供済み**（「↓」「分岐①」「Yes →」を含む） → A-3へ（粒度確認のみ）
- **業務説明あり** → A-2へ
- **何もなし** → A-1へ

粒度をask_user_inputで選択させる。

```
1. 経営層向け（概要レベル）
2. 業務担当・PM向け（要件定義レベル）← デフォルト
3. エンジニア向け（実装レベル）
```

**粒度合意なしに次のステップに進むことを禁止する。**

### A-1: ヒアリング（必要時のみ。5問以内）

1. このフロー図は誰が見ますか？
2. 何の業務の流れを描きますか？（「〜が完了するまで」で答えてもらう）
3. 登場する人・システムは？
4. 判断が発生するポイントは？
5. どこで始まりどこで終わりますか？

回答が曖昧な場合、具体例を挙げて再質問する。推測で補わない。
全問投げず、回答に応じて質問を絞る。

### A-2: テキストフロー生成

`references/text-flow-template.md` の標準フォーマットに従って構造化する。
**標準フォーマット以外の形式で出力することを禁止する。**

全ノードに以下を含める（粒度に応じた詳細度で）：
- 「誰が」（レーンで表現）
- 「何で」（補足テキストで実施場所/システム）
- 「何をする」（動詞で終わる）

### A-3: 分岐設計の最適化

全分岐について：
1. Yes↓が正常系・多数派になっているか → なっていなければ問いの向きを逆にする
2. No→の先が2ステップ以上あるか → 列展開を示唆
3. 複数の分岐が同じフローに合流するか → 参照ノード（同色）の使用を示唆

### A-4: 深掘り判定（省略禁止）

**全ノードについて1つずつ** `references/granularity-master.md` の該当列と照合する：

1. 粒度が適正か（粗すぎ/細かすぎでないか）
2. 3要素（誰が/何で/何をする）が揃っているか
3. このノードを分解しないと後続の分岐判断が曖昧になるか → Yes=ユーザーに確認
4. 粒度の混在がないか
5. 曖昧表現（〜等/〜など/仮/未定/TBD）→ 要確認事項に追加

### A-5: ユーザー確認（省略禁止）

テキストフローを提示し、ユーザーのOKを取る。
**OKなしにA-6に進むことを禁止する。**

### A-6: mdファイル出力

テキストフロー.mdを書き出す。収録内容：
1. メタ情報（フロー名/目的/粒度/レーン）
2. フロー構造（標準フォーマット）
3. 設計判断メモ（空の場合はなぜ空かを記載）
4. 要確認事項

present_filesで提示する。
出力後、「描画モードでHTML/SVGに変換できます」と案内する。

## モードB: 描画モード

テキストフローからHTML/SVGを生成する。または既存HTMLのレビュー・修正を行う。

**生成時は3ファイルを読む**: `template.md` → `style-guide.md` → `quality-check.md`
**レビュー時**: `quality-check.md` を先に読み、違反修正時に `style-guide.md` を参照

### B-1: テキストフローの受け取り

テキストフロー.md（設計モードの出力 or ユーザー直接提供）を受け取る。
標準フォーマットでない場合、設計モードの使用を案内する。

テキストフローから読み取る：
- アクター（レーン構成）
- 処理ステップ（ノード一覧）
- 分岐ポイント（ダイヤモンド一覧）
- 列構成（No→の先が2ステップ以上 → 列展開）
- 参照ノード（同色ノード）
- 開始/終了
- **Phase帯の有無を確認。ありの場合、`style-guide.md` §10を読む。**

分岐設計は設計モードで確定済み。受け取った構造をそのまま描画する。

### B-2: テキスト計算表の作成

`style-guide.md` §6-1〜§6-3に従い、全ノードのテキストを列挙。
全角/半角文字数を数え、テキスト幅と高さを計算する。
短い枝を含む全ノードを計算表に含める。
**⚠ 計算表が完成するまでB-3に進めない。**

### B-3: 図形サイズ表の作成

`style-guide.md` §6-4〜§6-7に従い、計算表から図形サイズを確定する。
最小サイズを適用する。同一列内の矩形幅を統一する。

### B-4: 座標表の作成

`style-guide.md` §7に従い、列構成とX座標を確定する。
全ノードのCY/上端/下端/左端/右端/合流Yを確定する。
前ノードの「下端」列+64pxで次ノードの上端を算出する。
**⚠ 座標表の全行が埋まるまでB-5に進めない。**

### B-5: 境界ボックス計算表の作成

`style-guide.md` §8に従い、座標表から境界ボックスを算出する。
レーン区切り線、右レーンCX、SVG幅/高さを計算する。
右レーンの全ノード座標を計算し、座標表に追記する。
境界ボックスを再計算する。

### B-6: SVG生成

`template.md` を参照して座標表の値をそのままSVGに転記する。
テキストは計算表のテキストをそのまま転記する。
矢印のline要素のみ、始点+6px、終点-14pxを適用する。
座標表にない値を使うことを禁止する。
計算表にないテキストを使うことを禁止する。

SVG生成中にテキストを修正したくなった場合、
SVG生成を中断し、B-2に戻る。1文字の修正でも例外なくB-2からやり直す。

### B-7: 最終走査

`style-guide.md` §9に従い、SVG生成後、全要素の右端・下端を走査する。
レーン干渉チェックを含む。
viewBox/width/heightを最終確定する。
**最終走査なしにpresent_filesを呼ぶことを禁止する。**

### B-8: セルフレビュー

`quality-check.md` の全チェック項目（A〜E）で自己検証する。
**最優先4項目（A-7, B-1, C-1, D-2）は省略不可。**
違反があれば修正してから出力する。

### B-9: 目視検証依頼

ユーザーにHTMLを提示し、以下を確認してもらう：
- 全ノードのテキストが図形内に収まっているか
- 右端・下端が見切れていないか
- 矢印のスペーシングが均一に見えるか

### B-10: ページ分割（オプション・ユーザー希望時のみ）

**トリガー**: ユーザーが「ページを分けて」「ページ区切りを入れて」「2ページに分割して」等を明示した場合のみ実行する。自動判断で実行しない。

**読むファイル**: `references/page-split.md`

page-split.mdの手順に従い、指定されたセクション区切りでHTMLを複数ページに分割する。

## モードC: 引き継ぎモード

作業の中間状態をhandover.mdに書き出す。

### C-1: フェーズ判定

| フェーズ | 状態 | 収録する内容 |
|---------|------|------------|
| 設計モード作業中 | テキストフロー設計中 | テキストフロー（作業中版）、設計判断、要確認事項 |
| 設計モード完了 | テキストフロー確定 | テキストフロー（確定版）、設計判断、要確認事項 |
| 描画モード作業中 | SVG生成中 | テキストフロー＋計算表＋座標表＋現在のHTML |
| 描画モード完了 | SVG確定 | 完成HTMLの参照 |

### C-2: handover.md作成

`references/handover-template.md` のフォーマットに従って書き出す。

収録内容：
1. 現在のフェーズ
2. テキストフロー（確定版 or 作業中版。未完了箇所は [未完了] マーク）
3. 設計判断メモ
4. 要確認事項
5. 計算表・座標表（描画モード作業中の場合のみ）
6. 次のセッションでやるべきこと（具体的なTodo）
7. 既知の問題点

### C-3: ファイル出力

handover.mdをpresent_filesで提示する。
「次のセッションでこのファイルをアップロードすれば作業を再開できます」と案内する。
